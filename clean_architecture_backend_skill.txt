## Core Architecture Rules (Enforce Strictly)

1. **Dependency Rule**: Domain and Application layers NEVER depend on Infrastructure or Presentation. Outer layers depend on inner via abstractions (Ports).
2. **SOLID Principles**:
   - Single Responsibility: One reason to change per class.
   - Open/Closed: Extend behavior without modifying existing code (new adapters).
   - Liskov Substitution: Ports must be substitutable.
   - Interface Segregation: Small, client-specific ports.
   - Dependency Inversion: Depend on abstractions, not concretions.
3. **Hexagonal / Clean Architecture**:
   - Core: Domain Entities + Use Cases (business rules).
   - Ports: Interfaces for repositories, external services.
   - Adapters: Concrete implementations (DB, API, Queue).
4. **Fully Asynchronous**: All database, HTTP, queue operations must be `async def` / `await`.
5. **No Business Logic** in controllers, routers, or repositories.

## Key Implementation Guidelines

### Domain Entities
```python
from dataclasses import dataclass
from datetime import datetime
from uuid import UUID

@dataclass(frozen=True)
class User:
    id: UUID
    email: str
    name: str
    is_active: bool = True
    created_at: datetime | None = None

    def activate(self) -> 'User':
        # Business rule example
        return User(**{**self.__dict__, 'is_active': True})
```

### Application Ports
```python
from abc import ABC, abstractmethod
from typing import Optional
from uuid import UUID
from domain.entities.user import User

class UserRepository(ABC):
    @abstractmethod
    async def get_by_id(self, user_id: UUID) -> Optional[User]: ...
    @abstractmethod
    async def save(self, user: User) -> User: ...
```

### Mappers
Always provide bidirectional mapping between Domain and Persistence/DTO layers.

### Use Cases
```python
from dataclasses import dataclass
from application.ports.repositories import UserRepository

@dataclass
class CreateUserInput:
    email: str
    name: str

class CreateUserUseCase:
    def __init__(self, repo: UserRepository):
        self.repo = repo

    async def execute(self, input_dto: CreateUserInput) -> User:
        # Domain logic / validation
        user = User(id=uuid.uuid4(), email=input_dto.email, name=input_dto.name)
        return await self.repo.save(user)
```

### Presentation (FastAPI)

Use `injector` library for automated DI. Bind ports to concrete adapters in container.

- Testcontainers or test DB for integration

### Async Processing
- Celery + Redis/RabbitMQ for background tasks
- Or aioredis for lighter needs
